# 童蒙家塾官网设计与建设背景文档

## 0. 文档用途
本文件用于让 Claude Code 快速理解“家庭 IP（童蒙家塾）”项目的官网建设背景、目标、约束与优先级，直接进入官网信息架构设计、页面设计与MVP开发。

## 1. 项目背景与业务目标
- 项目起点：妈妈长期做“家庭教育 + 传统文化”内容，目前主要发布在微信视频号，已积累内容与粉丝。
- 当前瓶颈：创作严重依赖妈妈个人精力，产能见顶；渠道单一；缺乏自有用户资产沉淀；个人IP形态不利于长期公司化协作。
- 长期方向：
  - 逐步从真人出镜过渡到虚拟IP（先轻虚拟：保留真人声音，画面虚拟化）。
  - 搭建主站沉淀内容与用户，再分发到多平台（抖音/小红书/B站等）。
  - 通过 AI Agent 自动化内容流水线提升产能。
  - 前台 IP，后台公司：保留真实教育理念，同时建立公司化品牌与合规体系。

## 2. 官网建设的核心目标（MVP优先）
- 目标1：把官网作为“内容源站”，沉淀结构化内容资产。
- 目标2：把官网作为“私域承接站”，完成留资/加企微/服务预约。
- 目标3：把官网作为“品牌与合规门面”，建立合作方与家长信任。
- 目标4：为后续 AI 自动化发布与多平台分发提供标准化数据源。

## 3. 目标用户与典型场景
- 核心用户：幼儿园至小学阶段孩子家庭，重视教育投入，关注习惯、品格、自主学习能力，不只盯分数。
- 用户诉求：
  - 短期可执行方法（能立刻在家实践）。
  - 长期系统方法论（可持续改善家庭教育效果）。
- 典型访问场景：
  - 从短视频平台跳转官网查看完整内容与资料。
  - 搜索某类教育问题（如习惯培养、亲子冲突）并找可执行方案。
  - 了解项目理念/团队背景，判断是否值得报名服务或合作。

## 4. 品牌定位与价值主张
- 项目名：童蒙家塾（作为官网统一主品牌名）。
- 核心定位：为家庭打造“孩子自学系统 + 家庭教育操作系统”。
- 差异化：不是零散育儿技巧，而是“真实家庭案例 + 方法论 + 可执行清单”的系统方案。
- 内容价值观基线：温和、正向、可执行，不羞辱、不贴标签、不神化孩子。

## 5. 官网信息架构（一级页面）
MVP 阶段建议采用清晰扁平结构：
1. 首页 Home
2. 栏目页 Columns
3. 内容详情页 Content
4. 旧视频号内容 Video Archive
5. 案例库 Case Library（MVP留空）
6. 关于我们 About
7. 合作联系 Contact
8. 法律声明 Legal

## 6. 页面级功能要求

### 6.1 首页（Home）
- 品牌首屏：一句话定位 + 核心价值 + 主要CTA（如“领取资料”“加入社群”）。
- 最新内容区：展示“图文为主”的卡片流，支持画廊/瀑布流切换（或二选一）。
- 栏目入口区：主理人理念、每日分享、家庭故事、教育方法、传统文化。
- 信任区：团队介绍摘要、理念摘要、媒体/合作信息（有则展示）。
- 旧视频号内容入口：保留独立入口卡片，跳转到 Video Archive 板块。
- 强制CTA：添加企微、订阅更新、预约咨询。

### 6.2 栏目页（Columns）
- 每个栏目独立列表页。
- 内容形态：图文卡片/画廊优先，视频仅作为补充信息出现。
- 支持筛选维度：主题标签、年龄段、场景标签。
- 支持排序：最新优先（MVP）、【待补充：是否需要热门排序】。

### 6.3 内容详情页（Content）
固定内容模板（强约束）：
1. 标题 + 摘要
2. 3-5条要点
3. 正文图文（主形态）+ 可选视频补充
4. 标签与栏目归属
5. 行动建议（可选）
6. 底部CTA（固定）

### 6.4 旧视频号内容（Video Archive）
- 目标：沉淀历史视频号内容，作为过往内容资料区，不干扰当前图文主内容流。
- MVP实现：先做列表占位 + 手动录入（标题、简介、发布日期、跳转链接）。
- 链接策略：若可导入原视频链接则直接外链；若暂不可导入，保留占位卡片与“即将补充”状态。
- 视觉形态：延续统一高级卡片风格，可加入封面图、平台角标与时间轴筛选。

### 6.5 案例库（Case Library）
- 当前策略：MVP阶段仅保留空态页面与结构，不放实际案例内容。
- 页面文案：展示“案例库建设中”与后续开放说明。
- 数据准备：保留后台字段结构，待后续逐步填充。

### 6.6 关于我们（About）
- 项目愿景、教育理念、核心团队角色。
- “前台IP，后台公司”的品牌策略说明。
- 【待补充：公司主体全称、工商/备案等正式信息】。

### 6.7 合作联系（Contact）
- B端/媒体合作表单。
- 项目介绍材料下载入口（PPT或PDF）。
- 【待补充：具体商务联系方式与响应SLA】。

### 6.8 法律声明（Legal）
至少包含：
- 未成年人保护原则
- 隐私政策
- 内容版权与授权声明
- 案例使用须知

## 9. 视觉与体验方向（官网UI）
- 设计目标：整体观感必须“高级”，避免模板化与普通内容站既视感。
- 整体气质：温暖、家庭感、可信赖、儿童友好、专业但不“教育说教化”。
- 视觉锚点：与IP内容统一，优先采用“3D黏土Q版（3d_clay_chibi）”资产做品牌识别。
- 色彩倾向：暖色系为主，避免高刺激对比与冰冷科技感。
- 语气风格：真实、谦逊、可执行，避免“神童”“赢在起跑线”等夸张词。
- 动效灵感（可选）：可参考“翻书式内容切换”交互（来自现有官网灵感笔记），用于强化故事感与记忆点。
- 高级感落地建议：
  - 使用大留白 + 强网格 + 精致排版层级（标题、导语、正文、注释清晰分层）。
  - 卡片采用轻阴影/细描边/材质感背景，避免廉价渐变和过度装饰。
  - 动效克制，重点用于页面切换和卡片入场，避免花哨。
  - 移动端保持同等质感，不做“桌面精致、移动降级”的割裂体验。
- 交互重点：
  - 首页和内容页的 CTA 必须高可见。
  - 内容检索与筛选路径要短。
  - 移动端优先（短视频导流场景占比高）。

## 10. 转化路径设计（必做）
官网内必须内嵌三类转化入口：
1. 资料领取：领取主题资料包/清单模板（导向留资或企微）。
2. 服务报名：预约咨询/带教申请。
3. 合作联系：机构合作、媒体合作入口。

转化漏斗目标：内容浏览 -> 信任建立 -> 留资/加企微 -> 后续服务转化。

## 11. 合规与风控边界（强约束）
- 未成年人信息保护：禁止暴露学校、住址、作息、可识别隐私等敏感信息。
- 宣传表述约束：禁止夸张神化儿童，强调方法可复制而非个体“天赋”。
- 案例发布前提：授权 + 匿名化（consent_status 必须可追踪）。
- 版权合规：外部素材需确认授权；官网需有侵权处理声明与流程。
- 舆情防护：预留侵权投诉与恶意剪辑应对说明页面/入口。

## 12. 技术与实施约束
- 人力现实：技术负责人（爸爸）每周可投入约20小时。
- 预算现实：MVP阶段以低成本为主（月预算几千人民币量级）。
- 实施原则：先最小闭环，后工程化升级。
- 当前相关工具生态：n8n / Dify / Seedance / CapCut（主要用于内容产线，可与官网数据结构衔接）。
- 【待补充：官网前后端技术栈最终拍板（如 Next.js + CMS / 其他）】。

## 13. MVP里程碑（官网侧）
- 第1周：
  - 上线官网基础框架。
  - 完成首页 + 至少3个栏目页框架 + 内容详情模板。
  - 完成图文卡片/画廊主样式（含移动端）。
  - 上线“旧视频号内容”占位板块（支持手动录入）。
  - 接入企微或表单CTA。
  - 上线法律声明基础页。
- 第2-3周：
  - 打通内容录入与发布流程（可先半自动）。
  - 完成分发文案包字段落地（distribution_pack）。
- 第4周：
  - 上线案例库空态页与后台预留字段（暂不填充实际案例）。
  - 上线内容风险检查清单（至少人工+规则提示）。

## 14. 非目标（MVP阶段暂不做）
- 全自动多平台 API 发布。
- 复杂多Agent状态机与大规模编排。
- 重度音视频生产后台（先保证官网内容沉淀与转化闭环）。
- 旧视频号链接自动抓取/自动导入（先手动录入，后续再评估自动化）。
